## Round 4 -- 연관 파일

1. DB 스키마를 변경했다면:
   - 기존 데이터와의 호환성이 유지되는가? (마이그레이션 필요 여부)
   - 해당 테이블을 참조하는 모든 Flask API 엔드포인트를 확인했는가?
   - crawl 스크립트의 INSERT/UPDATE 쿼리가 새 스키마와 일치하는가?
2. Flask 라우트를 추가/변경했다면:
   - 프론트엔드 JS에서 호출하는 URL이 일치하는가?
   - 응답 JSON 구조가 프론트엔드 기대와 일치하는가?
3. 팀/선수 데이터를 변경했다면:
   - TEAMS 리스트(main.py)의 id/sofascore_id가 일치하는가?
   - kleague_players_2026.json의 구조가 유지되는가?
   - default squad 파일 생성 로직에 영향이 없는가?
4. CSS/JS를 변경했다면:
   - 다른 페이지/컴포넌트에 의도치 않은 스타일 변경이 없는가?
   - Canvas 렌더링(전술판, 히트맵)에 영향이 없는가?
5. crawl 스크립트를 변경했다면:
   - update_data.py의 통합 실행 순서에 영향이 없는가?
   - 증분 수집 로직(이미 수집된 데이터 스킵)이 유지되는가?
