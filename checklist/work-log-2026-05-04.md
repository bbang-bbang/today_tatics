## 6. 핵심 발견 / 교훈

1. **SQL의 `SUM(CASE WHEN ... ELSE 0)` 함정**
   - 의도: 조건 안 맞으면 0
   - 실제: 모든 행이 NULL이어도 SUM=0 반환 → 코드의 `is None` fallback이 발동 안 함
   - 해결: `WHEN ... THEN expected_goals END` (ELSE 절 없음) 형태로 NULL 보존

2. **SofaScore ID vs K리그 공식 API ID 분리**
   - K리그 공식 API: 스코어 빠르게 (실시간), 단 lineup/incidents 없음
   - SofaScore: 풍부한 데이터, 단 K리그 공식 API와 ID 체계 다름
   - 같은 매치를 양쪽에서 다 받으려면 (date, teams) 키로 매핑 필요

3. **public IP 노출 + 비번 SSH = 24시간 봇 표적**
   - 도메인/IP 노출은 막을 수 없음 (포트 스캔 5초)
   - 진짜 방어는 자격증명 강화 (key only + fail2ban)
   - history rewrite는 운영 리스크만 키우고 보안 효과는 거의 없음

4. **UI 정보 중복 = 인지 부담**
   - 같은 "정확도"를 라운드 패널 chip + 매치 상세 bar chart 두 곳에 표시 → 사용자 혼란
   - canonical 1곳으로 정리 (PM 결정)
