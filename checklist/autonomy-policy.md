# Today Tactics 자율 판단 리스크 매트릭스
> 최종 수정: 2026-04-13
> 대상 프로젝트: K리그 전술 분석 웹 애플리케이션 (Flask + SQLite + JS)

## 원칙
다음 매트릭스를 기준으로 사용자 확인 없이 자율 실행할지, 확인을 요청할지 결정한다.
**사용자의 명시적 사전 허가("묻지 말고 실행해")가 있으면 Yellow Mid도 자율 실행 가능.**

---

## Red High -- 항상 확인

작업 전 반드시 사용자에게 의도와 영향 범위를 설명하고 승인을 받는다.

| 도메인 | 예시 |
|--------|------|
| DB 스키마 변경 | `players.db` 테이블 구조 변경, 컬럼 추가/삭제, 마이그레이션 |
| 팀/선수 마스터 데이터 | TEAMS 리스트(main.py) 수정, kleague_players_2026.json 구조 변경 |
| crawl 스크립트 핵심 로직 | SofaScore API 엔드포인트 변경, 증분 수집 로직 변경, 데이터 파싱 구조 변경 |
| 전술판 핵심 기능 | Canvas 렌더링 엔진, 화살표/드래그 코어 로직, 저장/불러오기 직렬화 형식 |
| Flask 라우팅 구조 | 기존 API 엔드포인트 URL 변경, 응답 JSON 구조 변경 (프론트엔드 호환성 파괴) |
| 외부 API 연동 | SofaScore/Open-Meteo/Nominatim API 호출 방식 변경 |
| 데이터 삭제 | DB 레코드 삭제, saves/squads 파일 삭제, 대량 데이터 변경 |

**이유**: DB 손상, 기존 저장 데이터 호환성 파괴, 외부 API 차단, 프론트엔드 동작 불능 등 서비스 장애로 이어질 수 있음.

---

## Yellow Mid -- 구두 확인 권장

작업 전 "~을 할 예정입니다"를 말하고, 사용자가 이의 없으면 진행한다.

| 도메인 | 예시 |
|--------|------|
| 새 Flask 라우트 추가 | 새로운 API 엔드포인트(/api/xxx) 추가 |
| 새 crawl 스크립트 추가 | 신규 데이터 소스 수집기 생성 |
| JS 주요 기능 추가 | 새로운 차트, 모달, 인터랙션 컴포넌트 추가 |
| 스탯 계산 로직 변경 | 90분 환산, 활동량 지수, 가중치 공식 변경 |
| 히트맵/Canvas 렌더링 변경 | 히트맵 시각화 방식 변경, Canvas 레이어 추가 |
| HTML 템플릿 구조 변경 | index.html 레이아웃 대폭 변경 |
| data/ JSON 파일 수정 | kleague_h2h.json, kleague_results_2026.json 등 데이터 파일 수정 |
| update_data.py 실행 순서 변경 | 통합 업데이트 파이프라인 단계 변경 |

**이유**: 연관 파일이 다수이고, 프론트엔드-백엔드-DB 정합성이 깨질 수 있음.

---

## Green Low -- 자율 실행

사용자에게 묻지 않고 즉시 실행한다. 완료 후 결과를 간략히 보고.

| 도메인 | 예시 |
|--------|------|
| CSS 스타일 수정 | 색상, 폰트, 여백, 레이아웃 미세 조정 |
| 텍스트/라벨 수정 | UI 문구 변경, 한글 표시명 수정 |
| 사소한 버그 수정 | NPE 방어 추가, 오타 수정, 조건문 보완 |
| 문서/주석 | 코드 주석, README, DEV_LOG, checklist 문서 갱신 |
| 정적 파일 추가 | 이미지(엠블럼 등), 아이콘 추가 |
| JS 미세 조정 | 기존 함수 내부 로직 소폭 수정 (API 호출/데이터 구조 변경 없음) |
| analysis/ 스크립트 | 분석 스크립트 수정/추가 (DB 쓰기 없이 읽기만 하는 경우) |

## 판단 기준 FAQ

**Q: 변경이 두 리스크 등급에 걸치면?**
A: 더 높은 등급을 따른다.

**Q: 긴급 수정이 필요한 Red High 항목이면?**
A: 상황을 설명하고 "지금 바로 수정해도 될까요?"를 물은 뒤 진행한다.

**Q: 사용자가 "알아서 해"라고 했으면?**
A: 해당 세션 한정으로 Yellow Mid까지 자율 실행. Red High는 여전히 보고 후 진행.

**Q: DB에 데이터를 쓰는 스크립트 실행이면?**
A: crawl/update 스크립트 실행은 Yellow Mid. DB 스키마 변경은 Red High.

**Q: 5인 관점 체크에서 한 명이라도 FAIL이면?**
A: 해당 FAIL 항목을 해결할 때까지 개발 완료로 판단하지 않는다. 리스크 등급과 무관하게 5인 전원 PASS가 필수 조건이다.

## 관점별 자율 판단 오버라이드

특정 관점이 관여하는 도메인은 해당 관점에서 리스크 등급이 상향될 수 있다.

| 관점 | 상향 대상 | 이유 |
|------|----------|------|
| P1 현직 감독 | 전술판 UI 변경을 Green -> Yellow로 상향 | 실전 활용에 직결 |
| P2 K리그 매니아 | 팀/선수 데이터 변경을 Yellow -> Red로 상향 | 팬 신뢰도에 직결 |
| P3 현업 선수 | 히트맵/개인 스탯 표시 변경을 Green -> Yellow로 상향 | 개인 역할 파악 정확도에 직결 |
| P4 스포츠 분석가 | 스탯 계산 공식 변경을 Yellow -> Red로 상향 | 분석 결론 신뢰도에 직결 |
| P5 전술 코치 | Canvas 렌더링 변경을 Yellow -> Red로 상향 | 전술 전달 정확도에 직결 |
| QA 엔지니어 | DB 쿼리/API 응답 구조 변경을 Green -> Yellow로 상향 | 30+ 엔드포인트 연쇄 회귀 버그 진원지 |
| 웹 디자이너 | CSS 레이아웃/컬러 시스템 변경을 Green -> Yellow로 상향 | 29팀 팀 컬러 정체성, 반응형 깨짐, 시각적 일관성에 직결 |
| 스포츠 도박맨 | 폼 지표/예측 알고리즘 변경을 Yellow -> Red로 상향 | 베팅 의사결정 근거 수치, 왜곡 시 실질적 피해로 이어짐 |
