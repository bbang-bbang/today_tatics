# Today Tactics 코드 리뷰 체크리스트
> 최종 수정: 2026-04-13
> 대상 프로젝트: K리그 전술 분석 웹 애플리케이션 (Flask + SQLite + JS)

## 진입 모드 결정
- **경량 모드**: 변경 파일 <= 2개 AND 동작 변경 없음 -> 1~4번 + 9번(5인 관점 체크)
- **전체 모드**: 그 외 모든 경우 -> 1~9번 전부 체크

---

## 1. 정확성 (경량 포함)
- [ ] SQLite 쿼리 결과가 None/빈 리스트일 때 안전하게 처리하는가?
- [ ] SofaScore API 응답 JSON의 null/누락 필드에 대해 방어 코드가 있는가?
- [ ] 날짜/시간 처리(KST, UTC 변환, timestamp)가 정확한가?
- [ ] 경기 데이터 증분 수집(이미 수집된 경기 스킵) 로직이 올바른가?
- [ ] Flask 라우트 응답(JSON/HTML)이 기대 구조와 일치하는가?

## 2. 타입 안전성 (경량 포함)
- [ ] API 응답에서 숫자형 필드(rating, goals 등)가 None일 때 0 또는 적절한 기본값으로 처리하는가?
- [ ] SQLite 컬럼 타입과 Python 코드에서 사용하는 타입이 일치하는가?
- [ ] JavaScript에서 서버 응답 JSON 파싱 시 타입 검증이 있는가?
- [ ] 좌표(lat/lon) 값의 유효 범위(한국: 위도 33~39, 경도 124~132) 검증이 있는가?
- [ ] 선수 스탯 90분 환산 계산 시 0분 출전(division by zero) 방어가 있는가?

## 3. 데이터 무결성 (경량 포함)
- [ ] DB 트랜잭션이 적절히 사용되고 있는가? (INSERT/UPDATE 묶음)
- [ ] 외래 키 관계(player_id, event_id, team_id)가 일관되게 유지되는가?
- [ ] 중복 데이터 삽입 방지(UPSERT 또는 존재 여부 체크)가 있는가?
- [ ] crawl 스크립트 중단 시 부분 데이터가 DB에 남지 않는가?
- [ ] JSON 파일(kleague_players_2026.json 등) 읽기 시 인코딩(UTF-8)이 보장되는가?

## 4. 보안 (경량 포함)
- [ ] SQL 인젝션 방어: 모든 쿼리가 파라미터 바인딩(?)을 사용하는가?
- [ ] Flask 라우트에서 사용자 입력(request.args, request.json)을 검증하는가?
- [ ] XSS 방어: 사용자 입력이 HTML에 직접 삽입되지 않는가?
- [ ] 파일 경로 조작(path traversal) 방어: saves/squads 디렉터리 외부 접근 불가한가?
- [ ] API 키나 민감 정보가 코드에 하드코딩되어 있지 않은가?

## 5. API 호출 (전체 모드)
- [ ] SofaScore API 호출에 적절한 딜레이/rate limit 처리가 있는가?
- [ ] HTTP 요청 타임아웃이 설정되어 있는가?
- [ ] API 응답 에러(4xx, 5xx)를 적절히 처리하는가?
- [ ] Open-Meteo API 호출 시 당일 데이터 미존재 케이스를 처리하는가?
- [ ] Nominatim API 초당 1건 제한을 준수하는가?
- [ ] Playwright 봇 탐지 우회가 안정적으로 동작하는가?

## 6. 성능 (전체 모드)
- [ ] SQLite 쿼리에 필요한 인덱스가 생성되어 있는가? (event_id, player_id 등)
- [ ] 대량 데이터(히트맵 158K+ 점) 조회 시 페이지네이션 또는 필터링이 있는가?
- [ ] Flask 응답에 불필요한 대량 데이터가 포함되지 않는가?
- [ ] 정적 파일(CSS/JS/이미지) 캐싱이 적절한가?
- [ ] crawl 스크립트가 불필요한 재수집(이미 있는 데이터) 없이 증분 수집하는가?

## 7. 완성도/엣지케이스 (전체 모드)
- [ ] K1/K2 리그 구분이 정확하게 처리되는가? (12팀 + 17팀)
- [ ] 시즌 교체(2024 -> 2025 -> 2026) 시 데이터 분리가 올바른가?
- [ ] 홈/어웨이 킷 색상이 겹치는 경우 구분이 가능한가?
- [ ] 경기장 좌표 lat/lon 뒤바뀜 자동 교정이 정상 동작하는가?
- [ ] 선수 이적/방출로 스쿼드 변경 시 기존 데이터와 충돌이 없는가?
- [ ] 브라우저 호환성(Chrome, Safari, Firefox)을 고려했는가?

## 8. UI/UX 정합성 (전체 모드)
- [ ] 전술판 드래그/드롭이 터치 디바이스에서도 동작하는가?
- [ ] 화살표(실선/점선/곡선/꺾기) 렌더링이 Canvas에서 정확한가?
- [ ] 팀 엠블럼/킷 아이콘이 올바른 팀에 매핑되는가?
- [ ] 애니메이션 재생이 프레임 드롭 없이 부드러운가?
- [ ] 히트맵 오버레이가 경기장 좌표계와 정확히 정렬되는가?
- [ ] 반응형 레이아웃이 다양한 화면 크기에서 깨지지 않는가?

## 9. 5인 관점 속성 체크 (경량 포함 -- 항상 수행)

모든 코드 리뷰에서 아래 5인의 관점으로 최종 점검한다.

### P1. 현직 감독 -- "실전에서 쓸 수 있는가?"
- [ ] 변경 사항이 실제 K리그 스쿼드 데이터와 정합성이 맞는가?
- [ ] 전술 스케치 도구의 핵심 기능(배치/화살표/애니메이션)에 영향이 없는가?
- [ ] 포메이션 변경/저장/불러오기가 정상 동작하는가?
- [ ] 경기 분석 데이터가 실전 의사결정에 활용 가능한 형태인가?

### P2. K리그 매니아 -- "팬 경험이 좋은가?"
- [ ] K1/K2 전체 29팀의 엠블럼, 팀 컬러, 선수 정보가 정확한가?
- [ ] H2H(상대전적) 데이터가 최신이고 정확한가?
- [ ] 경기 일정/결과 정보가 실제와 일치하는가?
- [ ] UI가 직관적이고 팬이 쉽게 탐색할 수 있는가?

### P3. 축구 현업 선수 -- "내 역할을 파악할 수 있는가?"
- [ ] 선수 개별 히트맵/스탯이 정확하게 표시되는가?
- [ ] 등번호/이름이 실제 선수와 일치하는가?
- [ ] 드래그 배치 후 전술 동선이 명확하게 보이는가?
- [ ] 모바일/태블릿에서 기본 기능이 사용 가능한가?

### P4. 스포츠 분석가 -- "데이터가 신뢰할 수 있는가?"
- [ ] 선수 스탯(레이팅, 패스, 태클 등)이 원본 소스(SofaScore)와 일치하는가?
- [ ] 90분 환산 지표 계산이 통계적으로 올바른가?
- [ ] 날씨/경기장 데이터가 실제와 부합하는가?
- [ ] 히트맵 좌표가 왜곡 없이 올바르게 매핑되는가?

### P5. 전술 코치 -- "분석 협업에 쓸 수 있는가?"
- [ ] A/B 양팀 동시 배치가 정상 동작하는가?
- [ ] 히트맵 오버레이가 상대 팀 분석에 유용한 형태인가?
- [ ] 전술판 저장/불러오기가 안정적인가?
- [ ] 다양한 화살표 타입이 전술 의도를 명확히 전달하는가?
